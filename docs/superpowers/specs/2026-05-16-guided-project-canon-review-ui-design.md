# MyNovel 项目、可信设定与章节审核引导式 UI 设计

日期：2026-05-16

相关资料：

- [MyNovel 聚焦创作台 UI/UX 重设计](2026-05-15-focused-creative-workbench-uiux-design.md)
- [MyNovel 可信设定提案 AI 修订设计](2026-05-13-canon-proposal-ai-revision-design.md)

## 1. 背景

当前 React 前端已经覆盖项目页、可信设定页和章节审核页，但三个页面都把管理信息、主要动作、状态详情和高级操作放在同一视觉层级里。

新手用户会遇到三个连续问题：

- 不知道当前项目下一步应该做什么。
- 不知道想改某部分内容时应该从哪里进入。
- 不知道修改、修订、批准之后会影响什么，尤其是否会写入可信设定。

本次设计目标不是增加更多解释文案，而是通过视觉层级、布局、动作权重和状态反馈，让用户自然理解流程。

## 2. 目标

- 把项目页、可信设定页和章节审核页串成同一条主流程：项目推进、设定修订、章节审核。
- 默认呈现新手友好的主路径，同时保留熟练用户所需的高级工具。
- 优先使用 AI 自然语言修订：用户描述修改意图，AI 生成预览，用户审核后应用。
- 所有会影响可信设定的动作，都必须先展示影响预览或状态变化摘要。
- 用视觉设计引导用户，不把页面做成说明书。

## 3. 非目标

- 不做完整新手教程、浮层引导或长段说明文案。
- 不把主交互改成聊天窗口。
- 不移除批量生产、导出、运行日志、手动编辑等高级能力。
- 不在本阶段重做服务端业务流程或数据模型。
- 不实现逐字段 diff，可先做分区级影响预览。

## 4. 设计原则

### 4.1 视觉先于说明

页面不能依赖长段文案告诉用户怎么用。主路径必须通过位置、尺寸、色彩、按钮权重、禁用状态和渐进展开自然显现。

允许使用的文字类型：

- 短标签。
- 按钮动作。
- 必要风险提示。
- 状态名称。

避免使用：

- 大段教学说明。
- 重复解释功能意义的说明卡。
- “你应该先...”这类生硬指令。

### 4.2 单页单主动作

每个页面默认只突出一个当前主动作。其他动作可以存在，但必须降级为次级按钮、菜单项、抽屉或高级视图。

主动作由当前状态决定：

- 项目页：运行当前章节、打开章节审核，或去调整可信设定。
- 可信设定页：生成修订预览，或在预览完成后应用修订。
- 章节审核页：让 AI 修订，或批准并写入可信设定。

### 4.3 先看影响，再做决定

任何会改变设定或章节状态的动作，都要在动作附近展示影响范围。影响预览不需要用长文解释，而应使用分区徽章、状态块、数量和风险色表达。

### 4.4 渐进披露

默认视图服务新手，隐藏非当前任务信息。熟练用户可以通过“高级视图”或抽屉展开：

- 完整章节队列。
- 批量生产。
- 目标字数。
- 导出。
- AI 运行记录。
- 卷纲。
- 手动修正文。
- 原始结构详情。

### 4.5 专业工作台视觉

采用 Data-First + Progressive Disclosure 的工作型 SaaS 风格。界面应安静、清晰、可长时间阅读。

避免：

- 营销式 hero。
- 过多装饰性卡片。
- 大面积同色系堆叠。
- 依赖渐变、装饰图形或说明文字制造层级。
- 卡片套卡片。

## 5. 统一信息架构

三个页面统一为四个视觉区域：

1. `项目身份条`：紧凑展示书名、章节或页面上下文、状态、Canon 版本。
2. `当前动作主区`：页面最大的视觉区域，只承载当前最重要任务。
3. `影响面板`：贴近主动作，展示操作后会影响什么。
4. `高级抽屉`：收纳管理型、调试型和熟练用户操作。

项目身份条替代当前大 hero。标题不再占据首屏主要空间。

## 6. 项目页设计

项目页定位为“项目驾驶台”，不是信息总览。

### 6.1 默认布局

首屏包含：

- 紧凑项目身份条：书名、题材、目标读者、项目状态、可信设定版本。
- 当前推进卡：页面视觉最强区域，展示当前章节或阻塞点。
- 影响面板：展示点击主动作后的结果。
- 可信设定状态条：世界、人物、伏笔、章节摘要等 3-4 个状态块。
- 高级抽屉：项目工具和详细资料。

### 6.2 当前推进卡

当前推进卡根据项目状态自动决定内容：

- 有可运行章节：展示章节号、标题、短摘要，主按钮为 `运行当前章节`。
- 有待审核章节：主按钮为 `打开章节审核`。
- 可信设定未锁定或不完整：主按钮为 `调整可信设定`。
- 没有待推进章节：展示下一步创建或检查状态的入口，但不同时展示多个同权按钮。

### 6.3 影响面板

影响面板用紧凑状态块展示：

- 是否会启动 AI。
- 是否会进入章节审核。
- 是否会直接写入可信设定。
- 是否需要人工批准。

例如运行当前章节时，影响面板应表达：

- 会生成章节候选。
- 不会直接写入可信设定。
- 完成后进入章节审核。

这些信息以状态块、图标和视觉权重表达，避免长段说明。

### 6.4 高级抽屉

默认收起以下内容：

- 批量生产。
- 目标字数。
- 导出 Markdown / JSON。
- 最近 AI 进度。
- 卷纲。
- 完整章节队列。

展开后保持现有能力，但视觉权重低于当前推进卡。

## 7. 可信设定页设计

可信设定页定位为“设定工作台”，核心是受控 AI 修订，不是字段编辑器。

### 7.1 默认布局

首屏包含：

- 紧凑项目身份条：书名、项目状态、Canon 版本、完整度。
- 分区地图：世界、人物、势力、地点、关系、伏笔、章节摘要、变化历史。
- 修改面板：选择分区后输入自然语言修订意图。
- 影响预览槽：未提交时为空态结构，生成后展示变更预览。
- 分区详情抽屉：点击分区后展开完整内容。

### 7.2 分区地图

分区卡只展示可扫读信息：

- 分区名称。
- 条目数。
- 锁定状态。
- 最近变更状态。

可修订分区视觉更活跃，已锁分区更沉稳。当前选中分区使用明确边缘高亮或顶部状态条，不依赖说明文案。

### 7.3 AI 修订流程

默认流程：

1. 用户选择一个分区。
2. 修改面板绑定该分区。
3. 用户输入一句自然语言修改意图。
4. 点击 `生成修订预览`。
5. 页面生成分区级影响预览，但不覆盖当前设定。
6. 用户在预览中选择 `应用修订` 或 `放弃`。

`应用修订` 只在可应用预览存在时出现。提交修改意图时不显示任何会让用户误以为直接覆盖设定的主按钮。

### 7.4 影响预览

修订预览展示：

- 将被替换的分区。
- 因锁定不会被修改的分区。
- 风险或冲突。
- 修订摘要。

预览以分区徽章、状态块和短列表为主。原始 JSON 放入高级区域。

### 7.5 锁定后的状态

当书籍进入生产锁定状态：

- 修改面板降级为只读。
- 分区地图仍可查看。
- 自由修订入口隐藏。
- 页面通过视觉状态指向章节审核流程，因为锁定后的可信设定只能通过章节审核写入新版本。

## 8. 章节审核页设计

章节审核页定位为“结果判定台”。用户先看章节结果和状态影响，再做审核决定。

### 8.1 默认布局

首屏包含：

- 紧凑章节身份条：书名、第几章、章节状态、Canon 版本。
- 结果摘要带：正文完成状态、审计状态、状态变化数量、重大变化状态。
- 正文阅读区：保持舒适阅读宽度。
- 审核决策面板：固定在正文旁边或移动端正文下方。
- 高级抽屉：阶段链路、运行 trace、相邻章节、手动修正文、导出。

### 8.2 审核决策面板

默认只突出与当前状态匹配的主要动作：

- 有高风险问题：`让 AI 修订` 是主按钮，批准动作降级或禁用。
- 无阻塞问题：`批准并写入可信设定` 是主按钮。
- 有重大状态变化：批准区必须显示确认控件，并贴近影响面板。

次级动作包括：

- 退回修订。
- 保存手动修正。
- 导出正文。

这些动作不应与主批准动作并列成同等权重。

### 8.3 影响预览

影响面板展示本次批准会写入哪些内容：

- 人物。
- 关系。
- 伏笔。
- 章节摘要。
- 世界规则。
- 其他状态变化。

如果没有状态变化，显示明确的空状态块，而不是长段说明。

如果存在重大变化，重大变化确认控件必须靠近影响预览，避免用户在页面底部误点批准。

### 8.4 运行中状态

章节运行中时：

- 结果摘要带切换为阶段进度。
- 正文阅读区可显示已有草稿或等待态，但审核动作不可用。
- 审核决策面板淡出或替换为等待状态。
- 不显示批准相关主按钮。

## 9. 组件与样式边界

实现时应优先拆出小组件，避免现有页面继续膨胀：

- `ProjectIdentityBar`
- `PrimaryActionPanel`
- `ImpactPanel`
- `AdvancedDisclosure`
- `CanonSectionMap`
- `CanonRevisionPreview`
- `ChapterResultStrip`
- `ChapterDecisionPanel`

样式应复用现有 React/Vite 前端，不引入新 UI 框架。必要时可新增局部 CSS 文件，但要避免继续扩大单个全局样式文件的职责。

## 10. 状态规则

### 10.1 项目页

- `draft` 且可信设定不完整：主动作指向可信设定。
- 可信设定已锁定且存在 planned / needs_revision 章节：主动作运行当前章节。
- 存在 awaiting_review 章节：主动作打开章节审核。
- 存在 running 章节：主动作打开章节页查看进度。

### 10.2 可信设定页

- 无选中分区：默认选择第一个可修订且未锁分区。
- 分区已锁：修改面板不可提交。
- 存在 pending revision：预览区成为主视觉焦点。
- revision running：预览区显示等待态，应用按钮隐藏。
- revision failed：预览区显示失败态和可重试入口。

### 10.3 章节审核页

- `planned` / `needs_revision`：主动作运行本章。
- `running`：显示阶段进度，审核动作不可用。
- `awaiting_review`：显示结果摘要和审核决策。
- `accepted`：显示只读结果，批准动作隐藏。

## 11. 测试范围

前端测试需要覆盖：

- 项目页根据不同 book / chapter / canon 状态展示不同主动作。
- 项目页高级抽屉默认收起，展开后可访问批量生产、导出和目标字数。
- 可信设定页分区选择会切换修改面板目标。
- 可信设定页提交修订只生成预览，不显示应用按钮直到有 pending revision。
- 可信设定页已锁分区不可提交修订。
- 章节审核页根据审计风险切换推荐主动作。
- 章节审核页展示状态变化影响分区。
- 有重大变化时批准必须经过确认控件。
- 运行中章节不显示批准动作。

## 12. 成功标准

- 新手进入项目页后，首屏能自然看到唯一主动作。
- 用户想修改设定时，可以从分区地图进入，不需要读完整设定内容。
- 用户提交修改意图后，能清楚看到只是生成预览，不会直接覆盖设定。
- 用户批准章节前，能看到会写入可信设定的影响范围。
- 熟练用户仍能展开高级工具完成批量、导出、手动修正和日志排查。
- 页面不依赖长段教学文案来解释流程。
